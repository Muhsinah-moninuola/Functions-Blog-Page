Understanding Python Functions

What are Functions?

A function is basically a block of code that performs a specific task. Python gives us built-in ones like print() or len(), and modules bring more (like math.sqrt()). The cool part is: you can define your own functions. It makes your code cleaner, easier to read, and less stressful to debug. It's like defining tasks for a set of people, each to their own task. Once the individual is called, they know it's time to work.

Another example is sharing house chores to members in the house; {Tola; Dishes, Kemi; Sweeping; Ope; washing}. See what I did there, I stored it like a dictionary. Back to the matter, once Ope is called, she knows its time to wash and so on.


Basic Syntax:


def function_name():
    print("Hello from a function")
function_name()
  • def ‚Üí defines the function
  • Function name ‚Üí e.g. greet
  • Parentheses () ‚Üí can hold parameters
  • Colon : ‚Üí marks the start of the body
  • The body ‚Üí indented instructions (the actual task)
  • Function call ‚Üí you must call the function for it to run

Parameters and Arguments

A parameter is the variable listed inside the parentheses in the function definition. An argument is the value that is sent to the function when it is called. Parameters are like placeholders in the chore list. Arguments are the actual people/things filling those spots. Imagine a lunch rotation: - Parameter: "Who will cook?" - Argument: "Kemi"

Pass by Value vs Pass by Reference

When passing arguments into functions, python either gives a copy (safe) or the real thing (modifiable) It basically means: ‚ÄúIf I give you something, are you allowed to change the original, or are you just holding a copy?‚Äù

  1. Pass by Value

    This one is like giving someone a photocopy of your passport. They can draw on it, tear it, or write anything on it, but your real passport at home? Untouched. Used for immutable types;

    • Numbers
    • Strings
    • Booleans
    • Tuples

    Even if the function tries to change them, the original stays the same.

  2. Pass by Reference

    This one is like handing over your actual backpack. If they put new books in or remove some, your real backpack is affected. No copies ‚Äî direct access! Used for mutable types:

    • Lists
    • Dictionaries
    • Sets
    • Custom objects

    If the function modifies these, the original gets updated too.

Types of Functions in Python

Positional (Required) Arguments

These arguments must be provided in the exact order defined in the function. Imagine standing in line at a bank: the first person gets served first, the second next, and so on. If you miss your spot or mix up the order, things don‚Äôt work as expected. In Python, you must supply these arguments exactly as required, otherwise it will raise an error.

Keyword arguments are named when you pass them into a function, so the order doesn‚Äôt matter. Think of filling out a labeled form: name="John", age=25. Python knows which value belongs to which parameter because of the labels, not the order.

Default arguments come with preset values in the function definition. If you don‚Äôt provide a value, Python uses the default. Imagine ordering a drink at a restaurant: if you don‚Äôt specify, water is served by default. But if you want juice instead, you can override the default value.

These arguments must be passed by position only. You cannot name them. Picture ordering by number at a fast food drive-thru: you say ‚Äú#3‚Äù and move on. You can‚Äôt describe it in words (‚ÄúI want a burger with fries‚Äù) even if you know what‚Äôs inside.

Keyword-Only Arguments

Keyword-only arguments must be passed by name. You can‚Äôt just pass the value without the parameter label. Imagine an online order requiring a coupon code: the site says ‚ÄúUse coupon code: ‚ÄòSAVE10‚Äô in the coupon box.‚Äù You must explicitly type the label and value ‚Äî just writing ‚ÄúSAVE10‚Äù alone won‚Äôt work.

Arbitrary (Variable-Length) Arguments

Sometimes, a function might need to accept any number of extra arguments. Python uses *args for extra unnamed positional arguments, and **kwargs for extra named arguments. Think of *args as a suitcase that can carry any number of extra clothes, and **kwargs as a shopping bag filled with labeled items like ‚Äúshoes=red, hat=blue.‚Äù This makes functions flexible and able to handle unexpected input.

Default Function Note: When using default values, remember that a positional argument cannot follow a keyword argument, but a keyword argument can follow a positional one. This ensures Python knows exactly which value goes where.
